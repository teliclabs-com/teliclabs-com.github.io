Common Questions

Does my data ever leave my AWS account?

No. The entire stack — LLM proxy, vector store, document ingestion, and application layer — runs inside your AWS account and VPC. Amazon Bedrock processes prompts inside AWS infrastructure. No data is routed through Telic Labs systems.

What AWS services are required?

The baseline deployment uses ECS Fargate, Amazon Bedrock, S3, CloudWatch, IAM, ACM, and a VPC. We recommend a dedicated AWS account for isolation, but can deploy into an existing account with appropriate guardrails.

Which LLMs are supported?

We deploy via Amazon Bedrock, which provides access to Claude (Anthropic), Llama, Titan, and other foundation models available in your region. The guardrail layer is model-agnostic and can be updated as new models become available.

What does "30-day post-deployment support" include?

After handoff you have 30 days of async support for issues directly related to the deployment (bugs, configuration questions, ingestion pipeline issues). It does not include custom feature development. The Managed Layer retainer covers ongoing needs beyond that window.

Can you work with my existing Terraform or AWS setup?

Yes. We can deploy into existing accounts and integrate with existing Terraform state backends, remote state, and CI/CD systems. A brief architecture review call is included before any engagement begins to surface any constraints.

Is this FIPS 140-2 compliant?

We design IAM, VPC, and compute configurations to align with FIPS 140-2 requirements where possible using AWS FIPS endpoints. Full FIPS compliance depends on your specific regulatory context — we include a compliance mapping document in every engagement so you know exactly where you stand.
